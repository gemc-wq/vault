## What & Why
Cem downloads Amazon active listings files weekly (6-9GB per region). We need to:
1. Store a parsed snapshot locally (SQLite on Mac Studio — fast, no upload limits)
2. Push only the **delta** (changes) to Supabase each week for Jay Mark + remote agents to query
3. Track shipping template compliance, new listings, delisted SKUs, price changes

This enables weekly automated audits without re-scanning 9GB files every time.

---

## Architecture

```
Weekly listings file (6-9GB .txt)
        ↓
  Python delta script (Mac Studio)
        ↓
  [1] Update SQLite snapshot (full catalog, local only)
        ↓
  [2] Compute delta vs previous snapshot
        ↓
  [3] Push delta rows to Supabase `listings_delta` table
        ↓
  Jay Mark queries Supabase remotely
  Prism/Ava query Supabase for gap analysis triggers
  Wednesday cron audits Supabase for shipping template issues
```

## SQLite Schema (local, Mac Studio)

### Table: `amazon_listings_snapshot`
```sql
CREATE TABLE amazon_listings_snapshot (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    region TEXT NOT NULL,              -- 'US', 'UK', 'DE'
    seller_sku TEXT NOT NULL,
    asin TEXT,
    design_code TEXT,                  -- parsed from SKU position [2]
    device_code TEXT,                  -- parsed from SKU position [1]
    product_type TEXT,                 -- parsed from SKU position [0], F prefix stripped
    item_name TEXT,
    price REAL,
    quantity INTEGER,
    open_date DATE,
    merchant_shipping_group TEXT,
    snapshot_date DATE NOT NULL,       -- date this file was loaded
    is_active INTEGER DEFAULT 1,       -- 0 = delisted in latest snapshot
    UNIQUE(region, seller_sku, snapshot_date)
);

CREATE INDEX idx_design ON amazon_listings_snapshot(design_code, region);
CREATE INDEX idx_snapshot ON amazon_listings_snapshot(snapshot_date, region);
CREATE INDEX idx_shipping ON amazon_listings_snapshot(merchant_shipping_group, region, snapshot_date);
```

### Table: `listing_snapshots_meta`
```sql
CREATE TABLE listing_snapshots_meta (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    region TEXT,
    snapshot_date DATE,
    filename TEXT,
    total_rows INTEGER,
    loaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Supabase Schema (remote, delta only)

### Table: `listings_delta`
```sql
CREATE TABLE IF NOT EXISTS public.listings_delta (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    region TEXT NOT NULL,
    seller_sku TEXT NOT NULL,
    asin TEXT,
    design_code TEXT,
    device_code TEXT,
    product_type TEXT,
    price REAL,
    quantity INTEGER,
    open_date DATE,
    merchant_shipping_group TEXT,
    snapshot_date DATE NOT NULL,
    change_type TEXT NOT NULL,    -- 'NEW', 'DELISTED', 'PRICE_CHANGE', 'TEMPLATE_CHANGE', 'QTY_CHANGE'
    old_value TEXT,               -- previous value (for change types)
    new_value TEXT,               -- new value (for change types)
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_delta_region ON listings_delta(region, snapshot_date);
CREATE INDEX idx_delta_design ON listings_delta(design_code, region);
CREATE INDEX idx_delta_change ON listings_delta(change_type, snapshot_date);
CREATE INDEX idx_delta_template ON listings_delta(merchant_shipping_group, change_type);
```

### Table: `listings_shipping_issues`
```sql
-- Materialised view of current wrong-template listings (refreshed weekly)
CREATE TABLE IF NOT EXISTS public.listings_shipping_issues (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    region TEXT,
    seller_sku TEXT,
    asin TEXT,
    design_code TEXT,
    current_template TEXT,
    correct_template TEXT,
    open_date DATE,
    days_listed INTEGER,
    first_detected DATE,
    resolved_date DATE,
    status TEXT DEFAULT 'OPEN'    -- 'OPEN', 'RESOLVED'
);
```

## Delta Script: `scripts/sync_listings_delta.py`

```python
#!/usr/bin/env python3
"""
Weekly Amazon listings delta sync.
Usage: python3 sync_listings_delta.py --region US --file ~/Downloads/US\ Active+Listings+Report_03-28-2026.txt
"""

import argparse
import sqlite3
import pandas as pd
import requests
from datetime import date
import os

# Config
SQLITE_PATH = os.path.expanduser("~/.openclaw/workspace/data/listings.db")
SUPA_URL = "https://auzjmawughepxbtpwuhe.supabase.co"
SUPA_KEY = "[REDACTED_JWT_PREFIX]..."  # service role key
HEADERS = {"apikey": SUPA_KEY, "Authorization": f"Bearer {SUPA_KEY}", "Content-Type": "application/json"}

CORRECT_TEMPLATES = {
    "US": "Reduced Shipping Template",
    "UK": "Nationwide Prime",
    "DE": None  # TBD
}

def parse_sku(sku):
    parts = str(sku).split('-')
    if len(parts) < 3:
        return None, None, None
    product_type = parts[0].lstrip('F') if parts[0].startswith('F') and parts[0] not in ['FLAG','FRND','FKFLOR'] and not parts[0].startswith('F1') else parts[0]
    device_code = parts[1] if len(parts) > 1 else None
    design_code = parts[2] if len(parts) > 2 else None
    return product_type, device_code, design_code

def load_listings_chunked(filepath, region, snapshot_date):
    """Load listings file in chunks, return parsed DataFrame."""
    cols = ['seller-sku', 'asin1', 'item-name', 'price', 'quantity', 'open-date', 'merchant-shipping-group']
    chunks = []
    for chunk in pd.read_csv(filepath, sep='\t', chunksize=50000, usecols=cols,
                              on_bad_lines='skip', encoding='latin-1', low_memory=False):
        chunk['region'] = region
        chunk['snapshot_date'] = snapshot_date
        parsed = chunk['seller-sku'].apply(lambda s: pd.Series(parse_sku(s), index=['product_type','device_code','design_code']))
        chunk = pd.concat([chunk, parsed], axis=1)
        chunks.append(chunk)
    df = pd.concat(chunks, ignore_index=True)
    df.columns = [c.replace('-','_') for c in df.columns]
    return df

def compute_delta(conn, new_df, region, snapshot_date):
    """Compare new snapshot against previous, return delta rows."""
    prev_date = conn.execute(
        "SELECT MAX(snapshot_date) FROM listing_snapshots_meta WHERE region=? AND snapshot_date<?",
        (region, snapshot_date)
    ).fetchone()[0]
    
    if not prev_date:
        # First run — everything is NEW
        new_df['change_type'] = 'NEW'
        new_df['old_value'] = None
        new_df['new_value'] = None
        return new_df
    
    prev_df = pd.read_sql(
        "SELECT seller_sku, price, merchant_shipping_group, quantity FROM amazon_listings_snapshot WHERE region=? AND snapshot_date=?",
        conn, params=(region, prev_date)
    )
    
    delta_rows = []
    prev_skus = set(prev_df['seller_sku'])
    new_skus = set(new_df['seller_sku'])
    
    # New listings
    for sku in new_skus - prev_skus:
        row = new_df[new_df['seller_sku'] == sku].iloc[0].to_dict()
        row['change_type'] = 'NEW'
        delta_rows.append(row)
    
    # Delisted
    for sku in prev_skus - new_skus:
        row = prev_df[prev_df['seller_sku'] == sku].iloc[0].to_dict()
        row['change_type'] = 'DELISTED'
        row['region'] = region
        row['snapshot_date'] = snapshot_date
        delta_rows.append(row)
    
    # Changes on existing listings
    merged = new_df.merge(prev_df, on='seller_sku', suffixes=('_new','_prev'))
    
    template_changes = merged[merged['merchant_shipping_group_new'] != merged['merchant_shipping_group_prev']]
    for _, row in template_changes.iterrows():
        delta_rows.append({**row.to_dict(), 'change_type': 'TEMPLATE_CHANGE',
                           'old_value': row['merchant_shipping_group_prev'],
                           'new_value': row['merchant_shipping_group_new']})
    
    price_changes = merged[abs(merged['price_new'] - merged['price_prev']) > 0.50]
    for _, row in price_changes.iterrows():
        delta_rows.append({**row.to_dict(), 'change_type': 'PRICE_CHANGE',
                           'old_value': str(row['price_prev']),
                           'new_value': str(row['price_new'])})
    
    return pd.DataFrame(delta_rows)

def push_to_supabase(delta_df):
    """Push delta rows to Supabase in batches."""
    cols = ['region','seller_sku','asin1','design_code','device_code','product_type',
            'price','quantity','open_date','merchant_shipping_group','snapshot_date',
            'change_type','old_value','new_value']
    
    batch_size = 500
    rows = delta_df[[c for c in cols if c in delta_df.columns]].to_dict('records')
    for i in range(0, len(rows), batch_size):
        batch = rows[i:i+batch_size]
        r = requests.post(f"{SUPA_URL}/rest/v1/listings_delta", headers=HEADERS, json=batch)
        if r.status_code not in (200, 201):
            print(f"Supabase error batch {i}: {r.text}")

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument('--region', required=True, choices=['US','UK','DE'])
    parser.add_argument('--file', required=True)
    args = parser.parse_args()

    snapshot_date = str(date.today())
    print(f"Loading {args.region} listings from {args.file}...")
    df = load_listings_chunked(args.file, args.region, snapshot_date)
    print(f"  Loaded {len(df):,} rows")

    conn = sqlite3.connect(SQLITE_PATH)
    delta = compute_delta(conn, df, args.region, snapshot_date)
    print(f"  Delta: {len(delta):,} changes")

    # Write to SQLite
    df.to_sql('amazon_listings_snapshot', conn, if_exists='append', index=False)
    conn.execute("INSERT INTO listing_snapshots_meta (region, snapshot_date, filename, total_rows) VALUES (?,?,?,?)",
                 (args.region, snapshot_date, args.file, len(df)))
    conn.commit()

    # Push delta to Supabase
    if len(delta) > 0:
        push_to_supabase(delta)
        print(f"  Pushed {len(delta):,} delta rows to Supabase")

    # Shipping template issues
    correct = CORRECT_TEMPLATES.get(args.region)
    if correct:
        wrong = df[df['merchant_shipping_group'] != correct][['seller_sku','asin1','design_code','merchant_shipping_group','open_date']]
        wrong['correct_template'] = correct
        wrong['region'] = args.region
        print(f"\n  ⚠️  {len(wrong):,} listings on wrong template in {args.region}")
        # Push to listings_shipping_issues (new issues only)
        # ... (add dedup against existing open issues)

    print("Done.")
```

## Inventory Join for Shipping Template Audit (CRITICAL)

**Nationwide Prime (UK) = same-day dispatch commitment. A listing must only be on this template if UK blank stock > 0.**

The join logic to check stock before flagging a template issue:

```
Listing SKU: HTPCR-IPH16PMAX-NARUICO-CR
  → product_type = HTPCR (strip leading F if FBA)
  → device_code  = IPH16PMAX

Blank inventory lookup:
  → Supabase blank_inventory WHERE item_code LIKE '%IPH16PMAX%' AND warehouse = 'UK'
  → OR check listing quantity column from Amazon inventory report (proxy: quantity > 0 = FBA stock exists)
```
