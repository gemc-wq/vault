### Printer Downtime & Site Routing:

**Downtime handling:**
- Printer down at a site → manual trigger to reroute orders to Philippines
- No automated failover currently

**Inventory-based routing:**
- Cron job checks inventory, flags items at zero stock
- Pre-defined table maps which inventory items print at which site
- Not every blank case is stocked at every site

**Site Equipment Capabilities (HARD-CODED CONSTRAINTS):**
- **UK & US (Florida):** Have skin printers (H89 codes) + sticker cutters
  - Skins and stickers ALWAYS print at UK or US, NEVER Philippines
- **Philippines:** Has UV printers for cases but NOT skin printers or laser cutters
  - Certain products ALWAYS print in Philippines only
- **UK:** Has some products that ALWAYS print there
- Each site has a fixed capability matrix → determines what CAN be printed where

**Three-Layer Routing Logic:**
1. **Layer 1 — Equipment capability:** Hard-coded product type → site mapping
2. **Layer 2 — Stock availability:** Cron checks inventory, zero stock → reroute
3. **Layer 3 — Manual overrides:** Holidays, peak season, printer downtime

**Waiting for:** Product type → site capability mapping table from Cem
