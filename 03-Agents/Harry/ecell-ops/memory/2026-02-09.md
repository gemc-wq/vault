## SKU Structure & Inventory Model (CRITICAL BUSINESS LOGIC)

### SKU Anatomy:
```
HC-IPH16-HPOTDH37-HOP
│   │      │       │
│   │      │       └── Design variant (digital file)
│   │      └────────── Design/Lineup code (digital file)
│   └───────────────── Device model (stockable unit)
└───────────────────── Product type (physical product)
```

- **Product Type** (first segment): Physical product stocked — e.g., HC (Hard Case), HTPCR (TPU Clear), HLBWH (Leather Book Wallet), HDMWH (Desk Mat), H8939 (PS5 skin)
- **Device Model** (second segment): The stockable unit — e.g., IPH16, IPH16PMAX, A165G, DS5CT
- **Design Code** (everything after): DIGITAL FILE ONLY — irrelevant for physical inventory
- For inventory: only Product Type + Device matters
- Design = print file applied to blank product at production time

### Print-on-Demand Model:
- Stock BLANK physical products (by product type + device)
- Order comes in → print design onto blank → ship
- Inventory = tracking blank products, NOT finished goods
- Replenishment from China for blank stock

### Inventory Management Needs:
- Track physical stock of blanks (product type + device)
- Book stock IN when purchasing from China
- Book stock OUT when orders fulfilled
- Record what's being made each day per site/location
- Currently on MySQL with self-calculation
- Cem will share current MySQL inventory database

## Image Generation Pipeline — Green Screen Decision

- Cem asked whether to use green screen on product templates for design placement
- **Decision: YES — use green screen templates**
- Green screen (#00FF00) provides exact mask for printable area
- Two approaches:
  1. **AI inpainting** — model replaces green area with design (clear boundary reduces hallucination)
  2. **Programmatic compositing** (RECOMMENDED) — chroma key → mask → perspective transform → composite. Deterministic, no hallucinations.
- Clear case templates are too ambiguous for AI placement (white/transparent = no boundary)
- Templates needed for each product type + device + angle
- Ava's project on iMac — reference images saved to `projects/design-automation/`
