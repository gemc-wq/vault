## Gateway EPERM Fix (morning)

**Problem:** Gateway was failing with `EPERM: operation not permitted, open C:\Users\gemc\.clawdbot\identity\device-auth.json`

**Root cause:** Clawdbot was originally installed as Administrator on the HP Cem PC (required admin to bypass Windows permission block during install). This caused the Scheduled Task and `device-auth.json` to be owned by Administrator, not `gemc`. Every subsequent gateway start inherited the wrong user context.

**Fix applied:**
1. `clawdbot doctor --fix` — removed invalid `allow` key from Telegram group config
2. Ran `clawdbot gateway stop` in regular (non-admin) PowerShell
3. Deleted `C:\Users\gemc\.clawdbot\identity\device-auth.json` (recreated fresh under `gemc`)
4. Ran `clawdbot gateway start` — RPC probe went green ✅
5. Deleted broken "Clawdbot Gateway" Scheduled Task from Task Scheduler (had mangled paths, no backslashes)
6. Re-registered with `clawdbot gateway install` from regular PowerShell

**Key lesson:** Always use regular (non-admin) PowerShell as `gemc` for all Clawdbot operations. Never run as Administrator — it corrupts file ownership and the gateway can't read its own auth files.

**Status:** Gateway fully operational, all providers online (Telegram, Discord, Slack, Canvas).
