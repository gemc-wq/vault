## Knowledge
- **Console Skin Replication Workflow**:
    1. Creative submits PSD + Google Sheet tracker via email.
    2. Replicator downloads assets and generates folder structure.
    3. Layouts are created in Smart Objects (do not rasterize).
    4. Naming convention: `[Index]_[Product_Name]` (e.g., `001_MIRACULOUS_LAD`).
    5. JSX script executes Photoshop actions (NEWSKIN, SERIESX, etc.).
    6. Automated export to designated folders followed by QC.
- **Product & Quality Identifiers**:
    - **Products**: `PS5SCS` (Standard), `PS5SDCS` (Slim Digital), `DS5EGCT` (Edge), `ASROGA` (ROG Ally), `STMDECK` (Steam Deck).
    - **Quality Tiers**: `CAT` (Catalog), `HPA` (High Print Accuracy), `LAD` (Listing Ad).
- **Current System Limitations**: Hardcoded Windows paths (prevents Mac compatibility), manual input requirements, lack of pre-processing validation, and no real-time progress visibility.
- **Proposed Automation Roadmap**: Transition to JSON-driven cross-platform config, Google Sheets integration for design codes, pre-flight validation, folder watching, and Slack/Telegram progress notifications.
