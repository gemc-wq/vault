## 2:42 PM — Royalty reporting deep dive findings
- Cem asked for a deep dive into old Zero/Xero royalty reporting logic to automate royalty reports from source systems instead of manual CSV export/import.
- **Core finding:** royalty logic is fragmented across 3 layers, not one system:
  1. **Contract master data** in old barcode DB tables: `t_royalty_information`, `t_royalty_information_temp`, `t_royalty_rates`.
  2. **Legacy royalty reporting engine** in PHP, especially `projects/zero-codebase/barcode/get_1d_sales.phpDREW042418`, which calculates net sales / GBP net sales, rewrites SKU + Description per licensor, and generates royalty report exports.
  3. **Newer JSON-configurable royalty app** in `/Users/openclaw/Downloads/zero/royalty-app/app/report_configs/` with per-licensor config files.
- **Key determination:** the royalty manager tables alone are NOT enough to replicate reporting. The licensor-specific logic lives partly in PHP hardcoding and partly in JSON + reference CSVs.
- **Territory conversions:** confirmed to exist explicitly in JSON report configs (examples observed: LFC, FCB, NFL, NHL, IM).
- **Net sales conversion:** confirmed in both legacy PHP and newer JSON rules. Legacy flow computes price minus tax -> net sales -> quantity extension -> GBP net sales via exchange-rate logic.
- **SKU/Description rules:** confirmed to be separate from Ava's normal SKU parsing rules. Legacy PHP contains hardcoded licensor-specific mapping logic for brands such as NFL, WWE, NBA, IMGC, One Direction, Outlander, Preacher, Steven Brown, etc.
- **Critical architectural insight:** royalty reporting uses a separate licensor-facing taxonomy. Internal SKU logic cannot simply be reused for royalty exports.
- **Reference/master SKU files:** newer royalty app also uses external master/reference CSV files for some licensors (example: LFC SKU master files), meaning some recognized licensor SKUs are not derived solely from code parsing.
- **Recommended future schema layers:**
  - `royalty_licenses`
  - `royalty_rates`
  - `royalty_territory_mappings`
  - `royalty_channel_mappings`
  - `royalty_sku_rules`
  - `royalty_description_rules`
  - `royalty_reference_skus`
  - `royalty_output_templates`
- **Recommended replication pipeline:** `orders -> licensor resolution -> territory mapping -> sku mapping -> description mapping -> net sales conversion -> royalty calc -> licensor-specific export`.
